# 06 — UI / UX

## Goal

Clear, professional flows for landing, auth, dashboard, upload, verify, history, and profile—without premature visual framework lock-in.

## Guidelines (Future Implementation)

- One primary action per screen (upload or verify).
- Show hash and transaction status in plain language.
- Prefer accessible forms and clear error states.
- **Do not introduce Tailwind until explicitly approved.**

`common/`, `navbar/`, `footer/`, `ui/`, `upload/` under `frontend/src/components/`.
